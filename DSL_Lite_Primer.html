Built for the Cyber Lakehouse
DSL Lite is a declarative, configuration-driven framework that transforms raw security logs from any source into OCSF-compliant, analytics-ready tables on Databricks — with no custom Spark code required. Security teams get a production-grade Bronze → Silver → Gold medallion pipeline in hours, not weeks or months. A reusable, scalable blueprint to build a Cyber Lakehouse.
Medallion Architecture
Bronze — Ingest & Amend Auto Loader streaming ingestion. Raw payload preserved as Variant or text with source, sourcetype, event time, dsl_id, and file metadata.
›
Silver — Parse & Curate Typed fields flattened from Variant or extracted as regex patterns into a structured schema. Filters, temporary columns, and business logic optionally applied.
›
Gold — Map & Normalize OCSF-compliant Delta Sinks with enforced schema standardization. Cross-vendor analytics on unified event tables with full source lineage tracking.
Key Features
  • Zero Spark code — declarative YAML presets define the full end-to-end pipeline
  • OCSF 1.7.0 compliance — unified Common Information Model across all vendors for cross-source security analytics
  • OCSF categories & events — Network, System, IAM, Findings, Application Activity · classes: DNS, HTTP, File System, Authentication, Process, API, and more
  • Source lineage tracking — source, sourcetype, and time carried through every layer
  • Multi-format ingestion — JSON, CSV, Syslog, and text via Auto Loader as Variant data type or text format
  • Streaming & batch — continuous or trigger-based pipeline execution
  • Schema enforcement — typed, versioned OCSF gold tables for performance and governance
  • Lookup enrichment — enrich events with Delta or CSV reference tables
  • Liquid Clustering — time-based clustering optimized for security queries
  • Idempotent ingestion — unique record_id per event prevents duplicate records across pipeline reruns
  • One-command deployment — Declarative Automation Bundles (DABs) per source
  • Agentic development — built-in agent skills for Databricks Genie and AI coding tools
Execution Modes
Spark Declarative Pipeline SDP
Lakeflow-managed pipeline. Serverless, continuous or triggered. Gold tables as Delta Sinks for cross-vendor consolidation.
Spark Structured Streaming SSS
Classic compute job. Three independent tasks (bronze / silver / gold) or a single medallion job. Full control over cluster configuration.
Streaming ingest (on request): Native file-based ingestion via Auto Loader is the default. Streaming source support — Apache Kafka, Databricks Zerobus Ingest, Azure Event Hubs, AWS Kinesis — are available as a development feature and can be enabled per engagement. Contact your Databricks team to scope.
